개인정보 처리방침

시행일: 2026년 2월 18일

PocketCode는 사용자의 개인정보를 외부 서버로 수집하거나 전송하지 않습니다. 모든 데이터는 사용자의 기기에만 저장되며, 앱 개발자는 이에 접근할 수 없습니다.

1. 수집하는 정보

PocketCode는 앱 기능 제공을 위해 다음 정보를 사용자의 기기 내에서만 처리합니다.

  • 호스트 구성 정보 — 사용자가 등록한 개발 서버의 이름, 주소, 포트, 연결 방식 등
  • 최근 폴더 기록 — 접속한 폴더 경로 및 마지막 접근 시간
  • 인증 정보 — GitHub 또는 Microsoft 계정의 OAuth 인증 쿠키 (WebKit에서 관리)
  • 연결 토큰 — serve-web 방식의 호스트에 필요한 접속 토큰 (iOS 키체인에 암호화 저장)
  • 앱 잠금 설정 — 생체 인증 사용 여부 및 잠금 타이밍 설정

2. 정보의 저장 위치

모든 데이터는 사용자의 기기 내에 저장되며, 외부 서버로 전송되지 않습니다.

  • 기기 내 데이터베이스 (SwiftData) — 호스트 목록, 최근 폴더
  • iOS 키체인 — 연결 토큰 (iOS에 의해 암호화)
  • WebKit 저장소 — 인증 쿠키, VS Code 에디터 캐시
  • UserDefaults — 앱 잠금 설정 등 환경 설정

3. 외부 서비스 연동

PocketCode는 다음 서비스에 직접 연결됩니다. 이는 앱의 핵심 기능을 위한 것이며, 분석이나 추적 목적이 아닙니다.

  • vscode.dev — VS Code 웹 에디터 인터페이스
  • github.com — GitHub 계정 인증
  • login.microsoftonline.com — Microsoft 계정 인증
  • 사용자가 등록한 개발 서버 — VS Code Tunnel 또는 serve-web 연결

PocketCode는 Firebase, Amplitude, Sentry 등 외부 분석 또는 오류 보고 SDK를 사용하지 않습니다.

4. 생체 인증

PocketCode는 앱 잠금 기능을 위해 Face ID 또는 Touch ID를 사용할 수 있습니다. 생체 인식 데이터는 iOS에 의해 처리되며, PocketCode는 생체 정보를 직접 저장하거나 접근하지 않습니다. 인증 결과(성공 또는 실패)만 앱 내에서 확인합니다.

5. 수집하지 않는 정보

PocketCode는 다음 정보를 수집하지 않습니다.

  • 위치 정보
  • 카메라 또는 마이크 데이터
  • 연락처, 사진, 캘린더
  • 광고 식별자 (IDFA)
  • 사용자가 작성하거나 편집하는 코드의 내용
  • 기기 사용 패턴 또는 행동 분석 데이터

6. 데이터 삭제

사용자는 언제든지 앱 내에서 데이터를 삭제할 수 있습니다.

  • 호스트 삭제 — 개별 호스트의 구성 정보, 연결 토큰, 최근 폴더 기록이 삭제됩니다.
  • 로그아웃 — GitHub 또는 Microsoft 인증 쿠키가 삭제됩니다.
  • 캐시 삭제 — 설정에서 WebKit 캐시를 삭제할 수 있습니다.
  • 앱 삭제 — 앱을 삭제하면 모든 데이터가 기기에서 완전히 제거됩니다.

7. 아동의 개인정보

PocketCode는 만 14세 미만 아동의 개인정보를 의도적으로 수집하지 않습니다.

8. 변경 사항

본 개인정보 처리방침은 필요에 따라 변경될 수 있습니다. 변경 시 이 페이지를 통해 공지하며, 시행일을 업데이트합니다.

9. 문의

개인정보 처리방침에 관한 문의 사항이 있으시면 GitHub를 통해 연락해 주세요.
